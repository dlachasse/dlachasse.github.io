Some projects I’ve worked on

To see (a limited selection) of things I’ve worked on you can visit my Github page or check out my blog One Drillion. Here are some of my projects with code samples.

Roasters.io

This is my most recent work and is a project I am using as a way to force myself into the habit of tdd. It allows users to create a running log of roasts they’ve tried by rating them on a simple 1-10 scale.

Repricer

This project began as a simple test to see if I could retrieve some pricing info from the Amazon Merchant api. Many merchants on Amazon will price their products with little to no margin. Sometimes this is due to mismanaged repricing software. So, I put in a little bit of logic to see if it’s worth it to lower our price to stay competitive with these other sellers. Here's a sample from it.

Fulfillment by Amazon reconciliation tool

I needed to figure out how to set a flag for products that run out-of-stock at Amazon’s fulfillment centers. So I created a tool to check daily for newly out-of-stock items. It can be found on Github.

Inventory Syncronizer

This is a tool I built to synchronize vendor inventory data with our local inventory. Before this was built, the process consisted of manually downloading inventory data from our vendor; formatting, verifying and combining local data; then uploading to Amazon.

The utility I built now accomplishes all of this thanks to Ruby and Amazon’s Merchant api. We have vastly decreased our number of backorders and are able to deliver on our in-stock promise to customers.

Previous work experience

I’ve spent the past 6 years working at a small online apparel (Blank Apparel) retailer selling exclusively on Amazon. I’ve had the chance to do things from printing t-shirts to designing, building and implementing a basic warehouse inventory system.

The position has provided me my first opportunity to program. When I first took the position I was tasked with building huge text inventory files in Excel. There were a number of tedious tasks that I wanted to speed up, and discovered automation with Visual Basic. I became obsessed with automation and optimizing broken workflows.

It was very empowering. I was learning new tools that could help me quickly accomplish tasks that would take an individual an inordinate amount of time. This encouraged me to learn even more and discover new uses for these newfound skills.

A little bit about me

Towards the beginning of 2013, I decided to invest my time into learning a programming language. Around the same time, a friend approached me about helping him build an open source bill of materials tool for 3D printing projects. So we spent some time learning php and building it together (link). Upon completing the project I wanted to take a step back and assess my learnings. ‘The language feels awkward, and the syntax makes the code so difficult to read!’ I thought to myself. Of course, I didn't consider the possibility that readability comes down to the code’s author. So I decided to try another language out.

I was primarily interested in developing for the web and had heard a little about Rails so I thought I would give it a shot. I found a couple books (rails tutorial and learn ruby the hard way) and gathered enough information begin building some of my own projects.

Since this, I’ve tried find other ways to improve my knowledge: I was able to get in to RailsConf when it came through Portland; I’ve gone to Ruby meetups/talks; I met up with developer friends to talk about projects and I try to pick up a technical book from time to time. I find that there is almost no limit to what one can learn when it comes to programming.
